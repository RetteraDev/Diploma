# Одолей Лихо
Проект представляет собой мобильную игру в жанре MMORPG с использованием геолокации для перемещения игрока

# Назначение системы
## Цель проекта
Цель проекта заключается в поддержании здорового образа жизни населения, борьбу с лишним весом, а также культурно-историческое просвещение игроков

# Требования к продукту

## Функциональные требования
1. Подготовка к игре
    1. Система должна уметь создавать новых и авторизовывать существующих пользователей в системе
    2. Система должна предоставлять возможность новым пользователям редактировать информацию о игровом аватаре бесплатно (установка никнейма, настройка внешнего вида персонажа)
    3. Система должна обучить нового игрока игровому процессу, показать основные механики, взаимодействие с интерфейсом пошагово
2. Настройка профиля
    1. Система должна предоставлять возможность всем пользователям редактировать информацию о игровом аватаре за внутриигровую валюту (установка никнейма, настройка внешнего вида персонажа)
    2. Система должна иметь возможность установить медицинские противопоказания игрока. В зависимости от указанных противопоказаний система должна показывать предупреждения о необходимости принять указанные лекарства, сделать специфичные действия
3. Игровой процесс
    1. Система должна уметь отправлять уведомления на мобильное устройство
    2. Система должна предоставлять каждому игроку обновленный список заданий ежедневно, в зависимости от его часового пояса в полночь
    3. Система должна предоставить каждому игроку список доступных наград, выделенных заказчиком в рамках договора или гранта
4. Геолокация
    1. Система должна уметь привязывать карту реального мира к игровому с использованием шестиугольных координат. Размер каждого шестиугольника ограничен максимальной точностью геопозициования в 5 метров
    2. Система должна уметь определять расположение парков и набережных. Такие места должны давать игроку за нахождение в этих шестиугольниках двойную награду
    3. Система должна уметь определять нечестных игроков, использующих для перемещения транспортное средство или VPN

## Нефункциональные требования

1. Приложение должно поддерживать устройства, работающие на операционных системах:
    - iOS 9.0- 16.0
    - Android 7.0 – 12.0 (учитывая специфические особенности марок Xiaomi, Sony, Huawei)
2. Пользователь, находясь в сети 3G, должен изменения в перемещении по карте с использованием геолокации не более чем за 2 секунды
3. Приложение должно взаимодействовать с сервером посредством протокола TLS/SSL
4. Сервер приложения должен уметь горизонтально масштабироваться по разным кластерам, взаимодействовать с репликами баз данных и поддерживать целостность данных
